# 과제 제출 가이드

## 기본 원칙
- 과제는 각자 별도 레포를 만드는 대신 이 레포 안에서 제출합니다.
- 본인 작업물은 `projects/<github-id>/` 아래에 정리합니다.
- 주차별 과제는 `week1`, `week2` 형태로 나눕니다.

## 제출 폴더 만들기
예시:

```text
projects/
  honggildong/
    week1/
    week2/
```

본인 GitHub 아이디가 `honggildong`이라면 `projects/honggildong/` 폴더를 사용하면 됩니다.

## 제출 순서
1. 이 레포를 `clone` 합니다.
2. 본인 폴더가 없다면 `projects/<github-id>/`를 만듭니다.
3. 해당 주차 과제를 `projects/<github-id>/week1/` 같은 위치에 정리합니다.
4. 변경 사항을 확인한 뒤 커밋합니다.
5. `push` 또는 운영자가 안내한 방식대로 `Pull Request`를 올립니다.

## 권장 커밋 메시지
- `Add week1 assignment`
- `Update week1 button text and colors`
- `Add personal project draft`

## 제출 전 체크리스트
- 본인 GitHub 아이디 폴더에 넣었는지 확인
- 실행에 필요한 파일이 빠지지 않았는지 확인
- 다른 사람 파일을 수정하지 않았는지 확인
- 변경 사항이 정상 실행되는지 확인

## 1주차 예시
- `week1/`의 예제 파일을 참고해서 본인 버전을 만듭니다.
- 이름, 만들고 싶은 사이트, 사용할 AI 도구, GitHub 아이디를 채웁니다.
- 버튼 문구, 색상, 클릭 시 문구를 원하는 내용으로 바꿉니다.
